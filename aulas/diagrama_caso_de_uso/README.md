# Caso de Uso

Ator representa os papeis assumidos no sistema(Exmplo: Cliente, Aluno, Professor). Inclui pessoas ou hardwares.

Caso de uso é uma ação que pode ser realizada por atores

## Relações

Um caso de uso é ligado para atores por meio de uma linha(associação). Mas é possivel ter relações entre casos e atores

### Casos

Inclusão: 
- faz o primeiro, faz o segundo obrigatorio. 
- Sempre que eu fizer algo, vou ter que fazer aquilo
- Usado quando existe complexidade em um caso, então se quebra em partes

Extenção:
- Nem Sempre que eu fizer algo, vou ter que fazer aquilo
- Usado quando existe complexidade em um caso, então se quebra em partes

Herança:
- Aproveita fluxo do caso de uso base

### Atores

Herança:
-  Um ator "x" que pode fazer tudo que outro ator "y" pode

## Esteriotipos

Rotulos.

Serve para caracterizar o diagrama.

## Caixa

Ilustra um package.

Serve para ilustrar modulos diferentes.

## Nota
- Serve para indicar detalhes ou condições, linkagem de nota a uma relação extend por exemplo.
- Pode ser associado a qualquer elemeno
